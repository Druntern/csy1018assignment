Web Developer, Games Developer and Programmer Developer

Web, Games and Programmer Developer jobs may seem all the same for someone who is not much into it, but if you understand about computing, you probably understand the big differences into each one of them.

First of all, a Web Developer doesn’t program. A Web Developer just builds the websites and maintain them as well with coding (e.g. html, css). Even between the web developers, there are the web programmers and the web designers. A web designer doesn’t even need to know how to code. They can just build mock-up designs using programs like Photoshop and Dreamweaver, while the web programmers are the ones who properly build the code inside the design to make the website work.

Then we have the programmers which have a totally different job as they deal with a bunch of other problems. A programmer has an enormous number of programming languages at disposal and their objective is to create software’s to ease many tasks in our every day life. Among the many languages, there are C, Java, Python, Swift, and lately Kotlin as the most known and useful.

Last of all, there are the games developers which in the beginning, every game developer needed to be a programmer, but nowadays, they are moving into scripting languages which basically shape the game itself. The scripts make the goals, conflicts, rules, balancing mechanisms, payoffs, rewards and messages. A game developer need to understand about geometry and physics, so the game can be realistic. They need to always keep up with the market, so the game has a good reception and good sales.

In the end, these jobs have a lot in common because they are all connected and every part of it runs in machines, within the same systems, but what differentiate them the most, is the different specializations each of one them are focused on.
